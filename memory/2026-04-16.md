## 📝 Lecciones Aprendidas

### Paperclip Device Authentication Pattern
1. DeviceId se calcula desde la private key del dispositivo
2. Dispositivos deben estar en `/root/.openclaw/devices/paired.json`
3. DeviceId debe coincidir exactamente entre:
   - Config de Paperclip (agent settings)
   - paired.json (gateway)
   - Device private key (hash SHA256)

### Heartbeat ≠ Status
- Heartbeats pueden llegar aunque Paperclip muestre "error"
- "error" puede ser por scopes/permisos, no por conexión
- Verificar logs del gateway para el error real

### Gateway Token Consistency
- Todos los agentes usan el MISMO gateway token
- El token está en `/root/.openclaw/gateway.token`
- Paperclip company config debe usar este token exacto
